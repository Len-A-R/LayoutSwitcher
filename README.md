# LayoutSwitcher

Минималистичная утилита для Windows, которая позволяет мгновенно конвертировать набранный текст в другую раскладку или менять его регистр — без необходимости стирать и перепечатывать всё заново.

## Возможности

* Конвертация раскладки — исправляет слово, набранное в неправильной раскладке (например, ghbdtn → привет)
* Смена регистра — верхний, нижний или инвертированный регистр выделенного текста
* Работа с выделением — если текст выделен, обрабатывается именно он
* Последнее набранное слово — если выделения нет, но вы только что печатали, обрабатывается последнее слово из буфера ввода
* Слово под курсором — если курсор стоит внутри слова, оно выделяется и обрабатывается автоматически
* Автозагрузка — опциональный запуск вместе с Windows
* Трей-иконка — отображает текущую раскладку клавиатуры (EN/RU)

## Горячие клавиши

| Комбинация                          | Действие                        |
| ----------------------------------- | ------------------------------- |
| `Shift` + `Shift` (двойное нажатие) | Конвертировать раскладку        |
| `LCtrl` + `LCtrl` (двойное нажатие) | ПРЕОБРАЗОВАТЬ В ВЕРХНИЙ РЕГИСТР |
| `RCtrl` + `RCtrl` (двойное нажатие) | преобразовать в нижний регистр  |
| `LCtrl` + `RCtrl` (одновременно)    | ИнвЕрТиРоВаТь РеГиСтР           |


Интервал между нажатиями: 500 мс

## Системные требования

* Windows 10 / Windows 11
* Поддерживаемые раскладки: Английская (US, 0x0409) и Русская (0x0419)

## Установка

1. Скомпилируйте проект в Delphi (RAD Studio)
2. Запустите полученный .exe
3. Приложение свернётся в системный трей
4. Через контекстное меню иконки в трее можно включить автозагрузку или выйти

## Технические детали

* Глобальный хук клавиатуры (WH_KEYBOARD_LL) — отслеживает нажатия без фокуса на окне приложения
* Глобальный хук мыши (WH_MOUSE_LL) — сбрасывает внутренний буфер при клике мышью
* Эмуляция ввода (keybd_event / SendInput) — имитирует Ctrl+C, Ctrl+V, Backspace и клики мыши для работы с любым редактором
* Буфер обмена — временно используется для получения выделенного текста, после операции восстанавливается в исходное состояние
* Cloud Clipboard — временные копии помечаются флагом ExcludeClipboardContentFromMonitorProcessing, чтобы не засорять историю Win+V

## Известные ограничения

* `HasSelectionInActiveWindow` работает только со стандартными Edit / RichEdit контролами; для браузеров и современных редакторов используется fallback через clipboard
* `GetWordAtCursor` использует двойной клик мышью — в некоторых приложениях (игры, полноэкранные режимы) это может работать некорректно
* Максимальная длина слова из буфера ввода: 50 символов
